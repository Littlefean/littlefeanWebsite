File类

概述：

主要用来操控电脑中的文件夹，在java.io包下

功能：

创建、删除、获取、判断是否存在、（文件或者文件夹），遍历文件夹，获取文件的大小。

重点：

file，directory，path

路径：

  1. static String pathSeparator 与系统有关的路径分隔符，为了方便，它表示为一个字符串【;】
  2. static char pathSeparatorChar 与系统有关的路径分隔符。
  3. static String separator 与系统有关的路径分隔符，为了方便，它表示为一个字符串【\】
  4. static String separatorChar 与系统有关的路径分隔符。

注意：

路径不要写死了。斜杠用File.sparator表示。

绝对路径和相对路径：

绝对路径：一个完整的根目录

盘符开始

相对路径：简化的路径

相对于当前项目的根目录

构造方法：

单个参数构造

File 对象名 = new File("C:\\\users...");

父路径和子路径构造

File(String parent, String child) 根据 parent 路径名字字符串和 child 路径字符串创建一个新file实例

获取功能方法：

  1. public String getAbsolutePath(); 返回此File的绝对路径名字符串
  2. public String getPath(); 将此file转换成路径名称字符串
  3. public String getName(); 返回此file表示的文件或目录名称
  4. public long length(); 返回此file表示此文件的长度

注意：文件夹是没有大小概念的

判断功能方法：

  1. public boolean exists(); 此File的文件或目录是否真实存在
  2. public boolean isDirectory(); 判断此File是否为文件夹结尾
  3. public boolean isFile(); 此File表示的是否为文件

创建删除功能方法：

  1. public boolean createNewFile(); 当且仅当具有该名称的文件尚不存在时，创建一个新的空文件

文件不存在，返回真并创建文件。

文件存在，返回假并不创建文件。

此方法抛出IO异常，我们调用了这个方法，就必须处理这个异常，要么throws，要么try catch

注意：此方法只能创建文件，不能创建文件夹。拆功能键文件的路径必须存在，否则会抛出异常

  1. public boolean delete(); 删除此File表示的文件或目录

举例：

  1. File f2 = new File("name\\\111\\\222\\\333\\\文件");
  2. boolean b2 = f2.delete();
  3. sout(b2);

注意：此方法直接删除，没有回收站

删除文件不成功的原因：

  1. 看看是否被别的进程引用，手工删除试试(删除不了就是被别的进程占用)
  2. file是文件夹 并且不为空，有别的文件夹或文件，
  3. 极有可能自己前面没有关闭此文件的流(我遇到的情况)
  1. public boolean mkdir(); 创建由此File表示的目录（创建单级文件夹）
  2. public boolean mkdirs(); 创建由此File表示的目录，包括任何必须但是不存在的目录（创建多级文件夹）

文件夹不存在，返回真并创建文件夹。

文件夹存在，返回假并不创建文件夹。

举例：

  1. File f2 = new File("name\\\111\\\222\\\333");
  2. boolean b2 = f2.mkdirs();
  3. sout(b2);

目录的遍历：

  1. public String[] list(); 返回一个String数组，表示该File目录中的所有子文件或目录
  2. public File[] listFiles(); 返回一个File数组，表示该File目录中的所有子文件或目录（打印出来会是绝对路径）

举例：

  1. File file = new File(...);
  2. String arr = file.list();
  3. for(String fileName : arr ){
  4. sout(fileName);
  5. }

注意：隐藏的文件也能获取到
