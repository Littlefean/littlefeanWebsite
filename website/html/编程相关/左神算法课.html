归并应用

求所有逆序对。利用了归并的稳定性。

归并、快排、堆排的比较

归并：【稳定性】稳定，但是空间占用O(N)

快排：不稳定，空间O(logN)，折中

堆排：【省空间】不稳定，（在堆化的过程中就已经不稳定了），但是占空间可以达到O(1)。

快排为什么比归并排序快一些？因为常数项低

看你在实际业务需求里要的是什么。

哈希函数与哈希表

由于有哈希均匀问题，所以可以考虑压缩哈希表

布隆过滤器（黑名单url）

把子节流设置成一个数组，用int类型去拼接，节省空间。

arr = [10101001010000101000111000110001100000000000000000]

有超级超级多的url，设计一个哈希表，只有add、in，功能。不考虑删除功能。

用多种哈希算法求不同的hash(url)，得到结果 index1,index2,index...indexk。

让arr[index1] = 1, arr[index2] = 1。。。这就是添加了。

检测某一个url是不是黑名单的时候，就把多种hash算法的结果都求出来，看看所有的对应索引位置上都是不是1，如果都是1，才是黑名单。

缺陷：可能会误杀，但是不会放过一个黑名单url。

只要arr开的够大，误杀率就会小。

k值，就是哈希函数的数量，取的越少，重叠率就越高，如果取的越多，那反而重叠率也越高，因为涂黑（标记为1）的arr位也就越多了，反而又增加了重叠的概率。
