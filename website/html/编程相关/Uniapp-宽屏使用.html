Uniapp宽屏适配

页面窗体级

分栏式宽屏适配

利用topWindow、leftWindow、topWindow

例如：在宽屏状态下，左侧做原生手机屏幕的界面（内容是目录级别的界面），右侧是详细信息的展示界面，右侧比左侧更宽，详细的图文内容更多，左侧像手机界面，右侧像平板界面，两个左右拼接在一起就是宽屏幕的电脑界面

注意：

  • topWindow、leftWindow、topWindow其中有一个存在的时候，topBar就会隐藏，不存在就不会隐藏
  • 目前的leftWindow、rightWindow、topWindow 只支持web端

需要修改 pages.json 文件 里的 leftWindow 或者rightWindow

  1. "rightWindow": {
  2.     "path": "responsive/right-window.vue",
  3.     "style": {
  4.       "width": "calc(100vw - 450px)"
  5.     },
  6.     "matchMedia": {
  7.       "minWidth": 768
  8.     }
  9. }

组件级适配

利用match-media组件、进行媒体查询

注意：

  • 快应用不支持这个组件
  • 支付宝小程序、qq小程序、百度小程序、字节小程序，暂不支持监听屏幕动态改变，即只执行一次媒体查询。

属性说明

属性名 类型 默认值 必填 说明
min-width number 否 页面最小宽度（ px 为单位）
max-width number 否 页面最大宽度（ px 为单位）
width number 否 页面宽度（ px 为单位）
min-height number 否 页面最小高度（ px 为单位）
max-height number 否 页面最大高度（ px 为单位）
height number 否 页面高度（ px 为单位）
orientation string 否 屏幕方向（ landscape 或 portrait ）
  1. <template>
  2.     <view>
  3.         <match-media :min-width="375" :max-width="800" >
  4.             <view>当页面最小宽度 375px， 页面宽度最大 800px 时显示</view>
  5.         </match-media>
  7.         <match-media :min-height="400" :orientation="landscape">
  8.             <view>当页面高度不小于 400px 且屏幕方向为纵向时展示这里</view>
  9.         </match-media>
  10.     </view>
  11. </template>

内容缩放拉伸

即：内容不会由于屏幕宽度的变化而现实或者消失，而是放大缩小

两种策略：局部拉伸与等比缩放

局部拉伸

有一部分是固定的，用的单位是px，另一部分是用的flex布局

等比缩放

根据页面屏幕宽度缩放，用的单位是rpx

对比

等比缩放比局部拉伸效果更省事，但是效果不好
