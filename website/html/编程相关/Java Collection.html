## 泛型

---

#### 概念：

​	未知数据类型，可以看作时一个变量，用来接收数据类型

#### 泛型通配符：

​	`ArrayList`

#### 泛型的上限：

​	格式：`类名称  对象名称`

​	意义：只能接收该类极其子类

#### 泛型的下限：

​	格式：`类型名称  对象名称`

​	意义：只能接收该类及其父类型



## 增强for循环

---

#### 概念：

​	专门用来遍历数组和集合，内部原理其实是一个Iterator迭代器，所以在遍历过程中不能进行增删操作

#### 格式：

```java
	for ( 元素数据类型 变量 ：Collection集合或者数组 ) {
		//操作
	}
```

# Set 

---

Set接口继承了Collection

Set接口特点：

1. 不允许元素重复
2. 没有索引，没有带索引的方法，也不能使用普通的for循环遍历

## HashSet

---

#### 特点

1. 不允许元素重复
2. 没有索引，没有带索引的方法，也不能使用普通的for循环遍历
3. 无序集合，存储元素和取出元素的顺序有可能不一致
4. 底层是一个哈希表结构（查询的速度非常快）

#### 操作实例：

```java
Set set = new HashSet<>();
set.add(1);
set.add(2);
set.add(45);
//使用迭代器遍历
Iterator it = set.iterator();
while(it.hasNext()){
    Integer n = it.next();
    System.out.println(n);
}
//使用增强for遍历
for (Integer i : set){
    System.out.println(i);
}
```

### HashSet 存储数据的结构（哈希表）

重地，通话，两个字符串的哈希值是一样的（巧合）

#### 介绍

JDK1.8版本前：哈希表 = 数组+ 链表

JKD1.8版本后：哈希表 = 数组 + 链表，或者，哈希表= 数组 + 红黑树

## 可变参数

#### 介绍

JDK1.5之后出现的新特性

#### 使用前提：

当参数列表的数据类型已经确定，但是参数的个数不确定，使用可变参数

#### 使用格式

```java
public static void test(int...varName){
    //方法体
}
```

#### 原理

可变参数底层是数组，根据参数传递个数不同，会创建不同长度的数组，来存储这些参数

传递参数的个数，可以01234……个

#### 举例

```java
public static int add(int...arr){
    int sum = 0;
    //遍历每个数组，获取每个数组里的元素
    for (int i : arr){
        //累加求和
        sum += i;
    }
    //返回求和结果
    return sum;
}
```

#### 注意事项

1. 一个方法的参数列表只能有一个可变参数
2. 如果方法的参数有多个，那么可变参数必须写在参数列表的末尾



# 数据结构

---

## 常见的数据结构

栈、队列、数组、链表、红黑树

### 栈

- 先进后出
- 入口和出口在同一侧

### 队列

- 先进先出
- 入口和出口在两侧

### 数组

特点：查询快，增删慢

查询快的原因：数组的地址是连续的我们通过数组的首地址可以找到数组，通过数组的索引可以快速查找某一个元素

增删慢的原因：数组的长度是固定的，想要增删就需要新创数组，并删除原有数组

### 链表

特点：查询慢，增删快

查询慢的原因：链表的地址不是连续的，每次查询都需要从头查询

增删快的原因：增删一个元素对整体结构没有影响，所以增删快。

结构：`自己的地址`--`数据`--`下一个节点的地址`

单项列表：链表中只有一条链子，不能保证元素的顺序（存储元素和取出元素的顺序有可能不一致）

双向链表：链表中有两条链子，其中一条专门记录元素的顺序，是一个有序的集合

### 树

#### 二叉树

分支不能超过两个

#### 排序树/查找树

在二叉树的基础上，元素是有大小顺序的，左子树小，右子树大

举例：猜数小游戏

#### 平衡树

左孩子和右孩子数相等

#### 红黑树

趋近于平衡树
