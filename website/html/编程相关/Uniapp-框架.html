uniapp框架学习

全局配置 pages.json

pages.json是全局文件，在里面进行全局配置

globalStyle

状态栏、导航条、标题、窗口背景色

（与pages里的style写法相同）

注意

  1. 支付宝小程序使用titleImage时必须使用https
  2. globalStyle中设置的titleImage也会覆盖掉pages->style内的设置文字标题
  3. 用 maxWidth 时，页面内fixed元素需要使用--window-left,--window-right来保证布局位置正确

topWindow、leftWindow、rightWindow

功能 属性 值 说明
配置页面路径 path 字符串类型
配置窗口表现 style 和pageStyle差不多
显示规则 matchMedia {"minWidth": 768} 是一个对象

pages

pages接收一个数组，数组里的每一个元素都是一个对象，该对象含有path和style属性

注意：

  • pages节点的第一项为应用入口页（即首页）
  • 应用中新增/减少页面，都需要对 pages 数组进行修改
  • 文件名不需要写后缀，框架会自动寻找路径下的页面资源

示例：pages.json文件里要添加的内容

  1. {
  2.     "pages": [
  3.         {
  4.             "path": "pages/index/index", 
  5.             "style": { ... }
  6.         }, {
  7.             "path": "pages/login/login", 
  8.             "style": { ... }
  9.         }
  10.     ]
  11. }

style

用于设置每个页面的状态栏、导航条、标题、窗口背景色，会覆盖 globalStyle 中相同的配置项

示例

  2.   "pages": [{

  4.       "style": {

  6.     },
  7.     ...
  8.   ]
  9. }

App.vue

uni-app的主组件，所有页面都是在App.vue下进行切换的，是页面入口文件。但App.vue本身不是页面，这里不能编写视图元素。

这个文件的作用包括：调用应用生命周期函数、配置全局样式、配置全局的存储globalData

main.js

uni-app的入口文件，主要作用是初始化vue实例、定义全局组件、使用需要的插件如vuex。

页面通讯

  1. uni.$emit

触发全局自定义事件

  1.    uni.$emit('update',{msg:'页面更新'})
  2.    
  1. uni.$on

监听全局自定义事件

  1.    uni.$on('update', function(data){
  2.        console.log('监听到事件来自 update ，携带参数 msg 为：' + data.msg);
  3.    })
  4.    
  1. uni.$once

监听全局的自定义事件。事件可以由 uni.$emit 触发，但是只触发一次，在第一次触发之后移除监听器。

  1.    uni.$once('update',function(data){
  2.        console.log('监听到事件来自 update ，携带参数 msg 为：' + data.msg);
  3.    })
  4.    
  1. uni.$off

移除全局自定义事件监听器

注意

  • 如果没有提供参数，则移除所有的事件监听器；
  • 如果只提供了事件，则移除该事件所有的监听器；
  • 如果同时提供了事件与回调，则只移除这个回调的监听器；
  • 提供的回调必须跟$on的回调为同一个才能移除这个回调的监听器；
