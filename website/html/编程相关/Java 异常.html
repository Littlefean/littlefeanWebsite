Throw关键字

解释

异常处理的第一种方式：交给别人处理

作用

在指定方法中抛出特定异常

格式

throw new xxxExcpetion ("异常产生的原因");

注意

  1. throw 关键字必须写在方法的声明处

  2. throw 后面new的对象必须是exception，或者其子类

  3. throw 关键字抛出指定异常对象，我们就必须处理这个异常对象

​ 两种处理方式：

​ throw关键字后边创建的是RuntimeException或者是其子类对象，可以不处理，交给JVM处理（打印异常）

​ throw关键字后边创建的是编译异常（写代码时候报错）我们就必须处理这个异常，要么抛出要么trycatch

参数非空判断：

  1. if(obj == null){
  2.     throw new NullPointerException("参数传递对象空");
  3. }
  5. Objects.requireNonNull(obj, "错误提示");

try...catch关键字

注意：

try中可能会抛出多个异常对像，使用多个catch来处理

格式

  1. try{
  2. //可能产生异常的代码 }catch(定义一个异常变量){
  3.     //异常处理 }

异常处理方法

Throwable类中定义了三个异常处理的方法

      举例：

      1. try{
      2.     readFile();
      3. }catch (IOException e){
      4.     System.out.println(e.getMessage());
      5.     e.printStackTrace();
      6.     return; //结束方法，为了不执行try以后的方法 }finally{
      7.     //无论是否出现异常都会执行     //注意：尽量不要在finally里面写return }

      子父类异常

      如果父类方法抛出了多个异常，子类重写这个方法时：

      ​ 抛出和父类相同的异常

      ​ 父类异常的子类

      ​ 不抛出异常

      父类方法没有抛出异常，子类重写该方法也不能抛出异常，只能捕获处理

      注意：

      父类异常什么样，子类异常就什么样

      自定义异常

      java提供的异常类不够我们使用需要我们提供一些异常类

      格式：

      1. public class XXXExpection extends Expection | RuntimeExpection{
      2. //添加空参构造方法     //添加一个带异常信息的构造方法 }

      注意：

      1. 自定义异常类一般都是以Exception结尾，说明是一个异常类

      2. 自定义异常类必须继承两者其一

      继承Exception：如果方法内部抛出了编译器异常，就必须处理这个异常，要么throws，要么try，catch

      继承RuntimeException：自定义的异常类就是一个运行期异常，无需处理，交给虚拟机处理
