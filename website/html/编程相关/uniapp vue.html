uniapp vue

与 传统vue的差异

  • 新增：uni-app 除了支持Vue实例的生命周期，还支持应用生命周期以及页面生命周期。
  • 受限：相比web平台，在小程序和App端部分功能受限，
  • uni-app 完整支持 Vue 模板语法。
  • App端可以使用更多的vue特性

模板语法

  1. <template>
  2.     <view>
  3.         <view>Message: {{ msg }}</view>
  4.     </view>
  5. </template>
  6. <script>
  7.     // 直接使用 es6 的语法来创建一个 vue的都西昂     export default {
  8.         // data 由 属性字典 变成了 方法         data() {
  9.             // 返回一个字典             return {
  10.                 msg: 'Hello Vue!'
  11.             }
  12.         }
  13.         // methods 不变         methods: {  
  14.             changetextvalue() {  
  15.                 this.textvalue="789"//这里修改textvalue的值，页面自动刷新为789               }
  16.      }  
  17.     }
  18. </script>

指令

v-bind 与传统类似，可以用 : 来缩写

  1. <image v-bind:src="imgUrl"></image>
  2. <!-- 缩写 -->
  3. <image :src="imgUrl"></image>

v-on 与传统一样，可以用修饰符 @ 来缩写

  1. <!-- 完整语法 -->
  2. <view v-on:click="doSomething">点击</view>    
  3. <!-- 缩写 -->
  4. <view @click="doSomething">点击</view>

v-once 与传统一样，保证只渲染一次 （h5 、小程序 不支持）

v-html 与传统一样，将标签内的内容设置为 该属性的字符串值，（h5 app 微信小程序均支持，微信小程序会转化为富文本）

data属性

必须直接让 data() 函数返回一个对象 这个对象不能用 const 声明，导致多个组件实例对象数据相互影响

class与style绑定

:class= 字典

与传统一样，v-bind绑定class对象，可以实时动态切换，也可以与普通的class共存

  1. <view class="已有普通类名" :class="{ 类名1: data里的布尔值, '类名2': data里的布尔值 }">
  2.     222
  3. </view>

:class= 数组

与传统一样，class=" 可以写成一个数组 "

  1. <view class="static" :class="[已有普通类名1 , 已有普通类名2]">
  2.     111
  3. </view>

可以像传统一样进行三元表达式的动态绑定

v-for

遍历字典

  1. 参数1：元素别名
  2. 参数2：键名
  3. 参数3：索引
  1. <view v-for="(value, name, index) in object">
  2.     {{ index }}. {{ name }}: {{ value }}
  3. </view>

和传统一样

希望每一项保持自己特征，需要使用 :key 来指定列表中项目的唯一的标识符。

在组件上使用

  1. <my-component v-for="item in items" :key="item.id"></my-component>

在组件上使用for的时候，key是必须要有的

事件处理

v-on 监听DOM事件

  1. <button v-on:事件名="执行代码 || 函数">
  2.     
  3. </button>

注意：如果是函数要写到methods里面

可以内置，也可以用 $event 和传统一样

事件修饰符

v-on 提供的:

  • .stop: 各平台均支持， 阻止事件冒泡，在非 H5 端同时也会阻止事件的默认行为
  • .native: 监听原生事件，仅 H5 支持
  • .prevent: 仅 H5 支持
  • .capture: 仅 H5 支持
  • .self: 仅 H5 支持
  • .once: 仅 H5 支持
  • .passive: 仅 H5 支持

注意：修饰符的顺序有讲究，用 v-on:click.prevent.self 会阻止所有的点击，而 v-on:click.self.prevent 只会阻止对元素自身的点击。

  8.         // 计算属性的 getter         reversedMessage(){
  9.             // 翻转字符串             return this.message.split('').reverse().join('')
  10.         }
  11.     }
  12. }

使用计算属性

  1. <view>Computed reversed message: "{{ reversedMessage }}"</view>

大括号里写computed属性里的方法名，不加小括号

setter

  1. <view>{{ fullName }}</view>

当手动修改属性的值时，触发setter函数，执行自己写的操作

  1. data() {
  2.     return {
  3.         firstName: 'Foo',
  4.         lastName: 'Bar'
  5.     }
  6. },
  7. computed: {
  8.     fullName: {
  9.         // getter         get(){
  10.             return this.firstName + ' ' + this.lastName
  11.         },
  12.         // setter         set(newValue){
  13.             var names = newValue.split(' ');
  14.             this.firstName = names[0];
  15.             this.lastName = names[names.length - 1];
  16.         }
  17.     }
  18. }

这个可以自己写名字，写好的名字会被解析，然后存到date里面，同时date里面的数据也可以展示出来，这就是setter和getter

计算属性能够进行响应式依赖进行缓存

使用场景：被多次依赖的时候可以用这个，带缓存的，减轻压力

监听器 watch

vue将会在实例化的时候调用$watch() ，遍历 watch 对象的每一个 property 。

  1. <template>
  2. <view>
  3. <input type="text" v-model="word">
  4.     </view>
  5. </template>
  6. <script>
  7.     export default {
  8.         data() {
  9.             return {
  10.                 word: 'word'
  11.             }
  12.         },
  13.         watch: {
  14.             /* 使用watch来响应数据的变化 */
  15.             word(newVal, oldVal) {
  16.                 console.log('最新值是：'+newVal,"原来的值是："+ oldVal);
  17.             }
  18.         },
  19.     }
  20. </script>
