暴力实验

环境：py3.7.2，pyCharm内运行同时开了百度云浏览器qq等其他应用程序

此次实验的意义在于让我们对python暴力的速度心里有个谱。

探究循环内print对程序执行速度的影响

  1. i = 0
  2. while True:
  3.     i += 1
  4.     print(i)

一分钟之后打印到了：15234500

如果改成这样

  1. def main():
  2.     i = 0
  3.     while True:
  4.         i += 1
  5.         if i % 100 == 0:
  6.             print(i)
  7.     return None

一分钟之后打印到了：588877600，速度是上一个的38倍

继续降低打印频率

  1. def main():
  2.     i = 0
  3.     while True:
  4.         i += 1
  5.         if i % 10000 == 0:
  6.             print(i)
  7.     return None

一分钟之后到了：761820000，速度是每一步都打印的50倍，总体提升的幅度减小了

再继续降低打印频率

  1. def main():
  2.     i = 0
  3.     while True:
  4.         i += 1
  5.         if i % 1000000 == 0:
  6.             print(i)
  7.     return None

一分钟之后到了：824000000，速度是第一次实验的54倍，速度提升效果已经不明显了。

一分钟可以遍历800..00（8后8个零） 这么多次

结论：

如果每一步都打印，会拖慢50倍左右

for循环比while循环快吗

  1. def main():
  2.     for i in range(1000000000000):
  3.         print(i)
  4.     return None

一分钟之后遍历到了：14596281，相比while循环，居然还慢了5%

列表推导式更快吗

  1. def timer(func):
  2.     def newFunc():
  3.         t1 = time()
  4.         func()
  5.         print(time() - t1)
  7.     return newFunc
  10. @timer
  11. def main():
  12.     arr = []
  13.     for i in range(500000):
  14.         arr.append(i ** 2)
  15.     return None
  1. 0.17652654647827148
  1. @timer
  2. def main():
  3.     # arr = []
  4.     # for i in range(500000):
  5.     #     arr.append(i ** 2)
  6.     arr = [i ** 2 for i in range(500000)]
  7.     return None
  1. 0.15957307815551758

只快了一点点 0.02秒，扩大数据量发现也依然只是快0.02秒

空for循环测试

  1. @timer
  2. def main():
  3.     for i in range(800000000):
  4.         pass
  5.     return None

13.081990480422974秒

14.064324378967285

如果里面添加一个判断语句

  1. @timer
  2. def main():
  3.     for i in range(800000000):
  4.         if i % 2 == 0:
  5.             pass
  6.         pass
  7.     return None

37.41891169548035，速度变慢了2.6倍

如果改成两个pass，是不是两个pass都会执行

  1. @timer
  2. def main():
  3.     for i in range(800000000):
  4.         pass
  5.         pass
  6.     return None

13.391043901443481秒，似乎不会

经过测试发现即使加了20个pass也还是13秒左右

如果把pass改成点点点

  1. @timer
  2. def main():
  3.     for i in range(800000000):
  4.         ...
  5.     return None

13.304449796676636，依然如此

多加了20行的点点点，还是13.175700902938843，13秒左右，貌似点点点还比pass更快。
