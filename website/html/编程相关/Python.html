​ 概念：为字符串中的每一个字符给定一个编号，相当于索引，火车票。

切片

概念：

截取其中的一部分

用法：

序列[开始位置下标：结束位置下标：步长]

特殊情况：

  • 不写开始，[：结束位置下标]，则从最开始开始。

  • 不写结束，[开始位置下标：]，则从开始位置到结束

  • 如都不写，[：]，则选区全部。

  • 步长负数，[：：-1]，将字符串倒叙输出

  • 位置负数，[-5：-1：]，从倒数第五个输出到倒数第一个，不包括倒数第一个。

注意：

  1. 步长选区间隔，默认为1，结束位置的下标是开区间，[ )

  2. 如果选取方向和步长方向冲突，则无法选取数据。

判断

  • startswith(): 检查字符串是否以指定子串开头，是返回真，否返回假。可指定范围检，设置开始和结束位置下标

​ 写法： startswith(子串，开始位置下标，结束位置下标)

  • endswith(): 检查字符串是否以指定子串结尾，是返真，否返假。可以指定范围检车，设置开始和结束位置下标

  • isalpha(): 如果字符串至少有一个字符并且所有字符都是字母，则反回真，否则返回假。

  • isdigit(): 如果字符串中只包含数字，则返回真。

  • isalnum(): 如果字符串至少有一个字符，并且所有字符都是字母或者数字则返回真

  • isspace(): 如果字符串只包含空白，则返回真。

函数参数详细写法

位置参数：

​ 解释：调用函数时根据函数定义的参数位置来传递参数

​ 注意：传递和定义参数的顺序和个数必须一致

​ 写法：

  1. def user(name, age, gender):
  2. print(f'您的名字是{name}，年龄是{age}，性别是{gender}')
  3. user('Tom',20,'男')

关键字参数：

​ 解释：函数调用，通过“键=值”形式加以制定。可让函数更加清晰、容易使用，同时也清楚了参数的顺序需求

​ 写法：

  1. def user(name, age, gender)
  2. print(f'您的名字是{name}，年龄是{age}，性别是{gender}')
  3. user('Rose', age=10, gener='女')
  4. user('小明'，gender='男', age=16)

​ 注意：函数调用时，如果有位置参数时，位置参数必须在关键字参数的前面，但关键字参数之间不存在先后顺序。

缺省参数（默认参数）：

​ 解释：也叫默认参数，用于定义函数，为参数提供默认值，调用函数时不可传该默认参数的值

​ 注意：

​ 所有位置参数必须出现在默认参数前，包括函数定义和调用

​ 函数调用时，如果为缺省参数传值，则修改默认参数值；否则使用这个默认值

​ 写法：

  1. def user(name, age, gender="男")
  2. print(f'您的名字是{name}，年龄是{age}，性别是{gender}')
  3. user('Tom', 20)
  4. user('Rose', 18, '女')

垃圾回收机制

作用：从经历过 引用计数器机制 但是仍未被释放的对象中找到 循环引用 干掉相关对象

底层机制

找到循环引用的步骤

  1. 收集所有 “容器对象” 通过一个双链表进行引用
  2. 针对每一个 “容器对象” 通过一个变量 gc_refs 来记录当前对应的引用个数
  3. 对于每个 “容器对象” ，找到它引用的 “容器对象” 并将这个“容器对象” 的引用计数 - 1
  4. 经历上述步骤后，如果一个容器的引用计数为0，代表它可以回收了

提升循环检测的性能

由于每次找循环引用非常消耗性能，所以python有一套机制来提升性能

产生假设：命越大的对象越长寿

假设一个对象10次检测都没有被干掉，那么这个对象一定很长久，就减少它的“检测频率”

分代回收机制

新增对象个数-消亡对象个数达到一定阈值时才会触发垃圾检测机制

  1. 默认一个对象创建出来之后都属于0代
  2. 如果经历过一代 “垃圾回收” 之后，依然存活，则规划到下一代
  3. 垃圾回收的顺序为：
  4. 0代“垃圾回收” 一定次数，会触发0代和1代回收
  5. 1代“垃圾回收” 一定次数，会触发0代1代2代税收

    类的设计原则

    遵循原则：SOLID

    1. S (Single responsbility principle) 单一职责原则

    一个类只负责一个功能

    好处：易于维护，高内聚代码，易于复用

    1. O (Open Close principle) 开放关闭原则

    对扩展开放，对修改关闭

    好处：易于维护，保证安全性

    1. L (Liskov Substitution principle) 里氏替换原则

    使用父类引用的地方必须能使用继承类的对象

    1. I (Interface Segreation principle) 接口分离原则

    如果类中包含了过多抽象方法，那么这些方法尽量分离

    1. D (Dependency Inversion principle) 依赖倒置原则

    高层模块不应该直接依赖低层模块、他们应该依赖抽象类或者接口
