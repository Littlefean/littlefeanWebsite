学习笔记 uniapp nvue

相比vue

  • native是本地的意思，性能更高
  • 组件更多

但是有些时候性能不如vue高：

  • canvas画布
  • 不支持媒体查询，动态横竖屏切换不方便

新建nvue界面

Hbuder：

文件 新建 uniapp页面 创建nvue界面

hbuder中不需要修改pages.json文件

开发页面

template

模板写法和数据绑定与vue相同

style

并非所有浏览器的css均支持，布局模型只支持flex布局

script

写法同vue，并且支持三种API：

  • nvue API
  • uni API
  • plus API

与vue的细节区别

  1. 只能用 v-if 不能用 v-show
  2. 默认竖排，改的话还得改manifest.json文件
  3. 文字内容必须写在 <text> 里面
  4. class进行绑定只支持数组写法
  5. 不支持使用ts

css方面

  1. 只能使用flex布局只能使用flex布局
  2. 布局没有百分比，没有媒体查询，横竖屏容易出问题，建议锁定方向
  3. 不支持背景图，但可以用 image 组件配合层级关系实现
  4. 只能使用class选择器，并且不能写选择器连接符
  5. 各组件在安卓默认透明，尽量设置 background-color 否则可能重影
  6. 安卓端尽量少用圆角，性能不好
  7. 不能在style中引入字体文件
  8. css不支持 1px red solid 这种简写

样式

盒子模型

和原生css盒子模型差不多

  • nvue盒模型的 box-sizing 默认为 border-box
  • 安卓nvue只支持 overflow:hidden
  • iOS 上，nvue支持 overflow:hidden 和 overflow:visible，默认是 overflow:visible

弹性盒模型

默认的，不需要手动为元素添加 display: flex; 属性

position定位

默认相对定位

定位只有：相对、绝对、固定、粘性

过渡动画

大部分和原生一样

变形

大部分也和原生一样

伪类

参数名 描述
active 所有组件都支持
focus 只有 input 组件和 textarea 组件支持
disabled 只有 input 组件和 textarea 组件支持
enabled 只有 input 组件和 textarea 组件支持

加了伪类的会覆盖不加伪类的

文字

只有 text 标签才能设置文字大小的功能，大部分和原生一样
