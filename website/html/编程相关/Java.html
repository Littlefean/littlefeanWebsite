静态关键字

静态成员变量

  1. public class Student{
  2.     private int age;
  3.     private String name;
  4.     
  5.     static String room; //可以统一修改     private static int idCounter = 0;  //可以用来做计数器     
  6.     public Student(String name, int age){
  7.         this.name = name;
  8.         this.age = age;
  9.         this.id = ++idCounter; //构造一个，计数器就加一个     }
  10. }

说明：

一旦使用static修饰成员方法，那么这就成为了静态方法。静态方法不属于对象，而是属于类的。

对于静态方法来说，可以通过对象名进行调用，也可以通过类名称来进行调用，但是更推荐类名称来进行调用，因为使用对象名调用容易让你误以为他是一个普通方法

注意：

  1. 静态方法不能直接访问非静态成员变量！因为在内存当中是先有的静态内容，后有的非静态内容

  2. 静态方法中不能用 this 。原因：this代表当前对象，通过谁调用的方法。谁就是当前对象。

构造方法的访问特点：

规则：

  1. 创建子类时，父类构造方法先执行，后执行子类构造方法。因为：子类构造方法中有一个默认隐含的super()调用，所以一定是先父后子的构造方法

  2. 父类构造方法有参数时，子类构造方法里面加一个super(对应的参数)。即：子类构造可以通过super关键字来调用父类重载构造

  1.    public class Zi{
  2.        public Fu(int n){
  3.            super(n); //这里            
  4.        }
  5.    }
  6.    

  1. super 的父类构造调用，必须是子类构造方法的第一个语句。不能一个子类构造调用多次super构造。

  2. 无参构造调用本类的有参构造时，使用this(对应参数); 但是只能调用父类或者子类的一种，不能无限套娃

总结：

​ 子类必须调用父类构造方法，不写则赠送super(); 写了则用写的指定的super调用，super只能有一个，还必须是第一个。

总结：

  1. java语言是单继承的

  2. java语言是多级继承

  3. 多个子类可以有相同的父类

      1. public class 实现类名称 implements 接口名称{
      2. ……
      3. }
      1. 接口的实现类必须覆盖重写接口中所有的抽象方法。

      ​ 实现：去掉abstract关键字，加上方法体大括号

      1. 创建实现类的对象，进行使用。

      注意事项：

      ​ 如果实现类没有覆盖重写接口种所有的抽象方法，那么这个实现类自己就必须是抽象类。

      接口默认方法 定义：

      简介：java8开始，默认方法被允许

      格式：

      1. public interface MyInterface{
      2.     public default 返回值类型 方法名称(参数列表){
      3. ……
      4. }
      5. }

      备注：接口当中默认的方法，可以解决接口升级的问题

      注意：

      1. 接口的默认方法，可以通过接口实现列对象，直接调用。

      2. 接口的默认方法，也可以被接口实现类进行覆盖重写。

      接口的静态方法定义：

      简介：java8开始，接口中允许定义静态方法。

      格式：

      1. public interface MyInterface{
      2.     public static 返回值类型 方法名称（参数列表）{
      3. ……
      4. }
      5. }

      提示：就是将abstract或者default换成static即可，带上方法体。

      注意：不能通过接口实现类对象来调用接口当中的静态方法。

      正确用法：通过接口名称，直接调用其中的静态方法。

      格式：接口名称.静态方法名(参数);

      接口的私有方法定义：

      背景：java9开始，接口当中允许定义私有方法。

      普通私有方法：解决多个默认方法之间代码重复问题

      静态私有方法：解决多个静态方法之间代码重复问题

      格式：

      ​ 普通私有方法：

      1. private 返回值类型 方法名称(参数列表){
      2. ……
      3. }

      ​ 静态私有方法：

      1. private static 返回值类型 方法名称(参数列表){
      2. ……
      3. }

      接口常量定义和使用：

      介绍：

      接口中也可以定义“成员变量”，但是必须使用public static final 三个关键字进行修饰。从效果上看，这其实就是接口的【常量】

      格式：public static final 数据类型 常量名称 = 数据值；

      注意：常量不能变，常量必须赋值，写法完全大写

      一个类多个接口的实现：

      格式：

      1. public class 实现类 implements 接口1， 接口2{
      2. ……
      3. }

      注意：

      • ​ 如果实现类所实现的多个接口中有重复的抽象方法，只需要覆盖重写一次即可。

      • ​ 如果有漏的抽象方法没覆盖，那么实现类必须是一个抽象类。

      • ​ 实现类一定要冲突的默认方法进行重写。

      • ​ 父类方法和接口的默认方法冲突了，父类优先。

      接口的多继承

      继承关系：

      1. 类 类，单继承

      2. 类 接口，多继承

      3. 接口 接口，多继承

      注意事项：

      1. 多个父接口当中的抽象方法如果重复，没关系

      2. 多个接口当中的默认方法如果重复，那么子接口必须进行默认方法的覆盖重写，而且带着defult关键字。

      多态

      介绍：

      ​ 代码当中体现多态性，其实就是一句话：父类引用指向子类对象

      格式：

      ​ 父类名称 对象名 = new 子类名称（）；

      ​ 或者

      ​ 接口名称 对象名 = new 实现类名称（）；

      口诀：

      ​ 看右边new的是谁就运行谁。

      多态中成员变量的特点：

      1. 直接通过对象名称访问成员变量：看等号左边是谁，优先用谁，没有则想上找。
      2. 间接通过成员方法访问成员变量：看该方法属于谁，优先用谁，没有则想上找。

      成员方法的使用特点：

      口诀：

      ​ 成员变量：编译看左边，运行还看左边。

      ​ 成员方法：编译看左边，运行看右边。

      多态的好处：

      new的左边看起来一样了。

      类的转型

      向上转型

      接口或父类名 新的对象名 = new 子类名（）；

      向下转型

      instanceof关键字转型

      注意：

      ​ 向下转型一定要用if判断，否则可能会出现异常。

      final关键字

      常见四种用法

      1. 修饰类

      2. 修饰方法

      3. 修饰局部变量

      4. 修饰成员变量

      final类

      写法：

      1. public final class 类名称{
      2. ……
      3. }

      含义：太监类

      final 成员方法

      写法：

      1. public final void method(){
      2. ……
      3. }

      含义：

      ​ 当final关键字用来修饰一个方法的时候，这个方法就是最终方法，也就是不能被覆盖重写。

      final 局部变量

      写法：

      ​ final int a = 50;

      含义：

      ​ 变量不可以被修改

      final 成员变量

      含义：不可变

      注意：

      1. 由于成员变量具有默认值，所以用了final之后必须手动赋值，不会再给默认值了。

      2. 对于final 的成员变量，要么使用直接赋值，要么通过构造方法赋值。

      3. 必须保证类当中所有重构方法，getter和setter，都会最终对final赋值不重复。

      四种权限修饰符

      比较：

      1. public > protected > (default) > private
      2. 同一个类 可访 可访 可访 可访
      3. 同一个包 可访 可访 可访 禁止
      4. 不同包子类 可访 可访 禁止 禁止
      5. 不同包非子类 可访 禁止 禁止 禁止

      内部类

      含义：

      如果一个事物的内部包含另一个事物，那么这就是一个类内部包含另一个类。

      例如：

      身体和心脏的关系、汽车和发动机的关系

      成员内部类

      定义格式：

      1. 修饰符：class 外部类名称{
      2. 修饰符 class 内部类名称{
      3. ……
      4. }
      5. }

      注意：内用外可以随意访问。外用内要借助内部类对象。

      内部类的使用：

      1. 间接方式：在外部类的方法当中，使用内部类；然后main只是调用外部类的方法。

      2. 直接方式：公式

      1. 类名称 对象名 = new 类名称（）；
      2. 外部类名称.内部类名称 对象名 = new 外部类名称（）.new 内部类名称（）；

      内部类的同名变量访问：

      1. 修饰符：class Outer{
      2. int num = 10;
      3. 修饰符 class Inner{
      4. int num = 20;
      5. public void method Inner(){
      6. int num = 30; //内部类方法的局部变量 sout(num); //局部变量，就近原则 sout(this.num); //外部类的成员变量 sout(Outer.this.num);//外部类变量的使用 }
      7. }
      8. }

      注意：如果出现了重名现象，那么格式是：外部类名称.this.外部类成员变量名

      局部内部类

      说明：如果一个类是定义在一个方法内部的，那么这就是一个局部内部类。

      “局部”：只有当前所属的方法才能使用它，出了这个方法外面就不饿能用了。

      定义格式：

      1. 修饰符 class 外部类名称{
      2. 修饰符 返回值类型 外部类方法名称（参数列表）{
      3. class 局部内部类名称{
      4. ……
      5. }
      6. }
      7. }

      注意：局部内部类，如果希望访问所在的方法的局部变量，那么这个局部变量必须是【有效final的】。

      原因：

      1. new出来的对象在堆内存当中

      2. 局部变量是跟着方法走的，在栈内存当中

      3. 方法运行结束后，立刻出栈，局部变量就会立刻消失。

      4. 但是new出来的对象回在对当中持续存在，知道垃圾回收消失。

      匿名内部类***

      说明：

      如果接口的实现类（或者是父类的子类）只需要使用唯一的一次，那么这种情况下就可以省略掉该定义，而改为使用【匿名内部类】

      格式：

      1. 接口名称 对象名 = new 接口名称（）{
      2. //覆盖重写所有抽象方法 }； //分号不能丢 对象名.成员方法名；

      注意：

      1. 在创建对象的时候，只能使用唯一一次，如果希望多次创建对象，而且类的内容一样的话，那么久必须使用单独定义的实现类了。

      2. 匿名对象在【调用方法】的时候，只能调用唯一一次，如果希望同一个对象调用两次方法，那么必须给对象起个名字。

      3. 匿名内部类是省略了【实现类/子类】，但是匿名对象是省略了【对象名称】
