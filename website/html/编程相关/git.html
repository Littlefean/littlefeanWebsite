学习笔记git

初始化git

  1. git config --global user.name "littlefean"
  2. git config --global user.email "....@123.com"

git中的文件结构

  • 未跟踪

  • 已跟踪

纳入了版本控制的文件，在上一次快照中有它们的记录

  • 已修改 modified
  • 已暂存 staged
  • 已提交 committed

设置工作区文件夹为仓库

  1. # 先在cmd种进入相应文件夹下
  2. git init
  3. # 会在该文件夹下生成.git隐藏文件夹，其中包括两部分，一个是暂存，另一个是分支

查看

  1. # 进入到路径下
  2. git status
  3. # 他会展示出未跟踪的文件
  5. git add .
  6. # 把工作区里的所有文件推进暂存区（跟踪某个文件）
  8. git commit -m "add to files"
  9. # 把暂存区的文件提交到版本库并添加一条说明信息
  11. git commit
  12. # Esc，退出输入状态。
  13. :wq
  14. # 写好之后输入:wq (我去)结束并提交

远程仓库

  1. # 我的远程仓库
  2.     https://github.com/Littlefean/EverydayEmail.git
  4. # 创建一个远程仓库
  5. git remote add 自己给远程远程仓库命名一个英文名 https://......
  7. # master是一个默认分值的名字
  8. git push 远程仓库英文名 master
  9. git push origin littlefean
  11. # 查看所有已有的远程仓库地址
  12. git remote -v
  14. # 下载文件
  15. git pull 远程仓库英文名 master
  16. git fetch https://github.com/Rutubet/CrAuto.git
  18. # 重叠仓库
  19. git checkout -b feature origin/feature
  20. # 注意：克隆仅仅在初始本地仓库的时候使用
  21. git clone https://......
  23. # 切换从远程仓库里克隆过来的一个分支
  24. # 因为默认是master分支，但是其他分支是不会显示出来的，所以要用下面的指令来进行切换分支
  25. git checkout --track origin/littlefean
  27. # 删除一个分支
  28. git branch -d littlefean

分支

主分支为master分支，这个一般是发布版的稳定的分支、然而一般开发的时候不会轻易往master分支上放东西，会往开发用的分支上放东西

  1. # 查看所有分支
  2. git branch
  4. # 创建分支
  5. git branch dev
  6. # 展示后前面带一个 * 的表示是当前的有效分支

切换分支

  1. git checkout 分支名

注意，切换到某一个分支的时候，工作区里的文件内容会发生改变

查看分支的提交记录

  1. # 查看简略模式的分支提交记录
  2. git log --oneline
  4. # 查看完整的详细的分支提交记录
  5. git log

分支合并

主要代码

  1. # 合并分支
  2. git merge 另一个分支的名字
  4. # 查看分支合并图形
  5. git log --online --graph

主要过程

  1. 提交当前的副分支
  2. 从副分支切到主分支上
  3. 把副分支的内容合并到主分支上

快速合并（fast-forward），触发条件：主分支没动，副分支动了，主分支上的指针直接移动到了副分支上

三方合并：绝大多数的状态

合并冲突

含有冲突的文件会出现以下内容

  1. <<<<<<< HEAD
  2. hello master(分支前的内容)
  3. ==============
  4. word dev update（另一个分支更改的内容）
  5. >>>>>>> Dev

过程：

  1. 合并文件
  2. 解决冲突
  3. 再次提交

取消跟踪

  1. git rm --cached a.txt

  5.    *.jar
  6.    *.war

弹出菜单, 方框全部勾选

将本地git仓库提交到远程

  1. 复制远程仓库地址
  2. 点击 "VCS" --- "Git" --- "push"
